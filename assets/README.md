# assets/

Static assets bundled into the Flutter app at build time.

---

## data/ps/

Pokémon Showdown data files used by the format engine (`lib/services/format/`). Generated by `scripts/sync_ps_data.py` — **do not edit by hand**.

| File | Source | Contents |
|------|--------|----------|
| `learnsets.json` | PS `learnsets.js` | Per-Pokémon move lists keyed by learn method and generation |
| `event_learnsets.json` | PS learnset data | Event and gift Pokémon movesets |
| `moves.json` | PS `moves.js` | Move stats: type, power, accuracy, PP, Z-move and Max move data |
| `items.json` | PS `items.js` | Item data: generation introduced, Mega stone → form, Z-crystal → move mappings |
| `abilities.json` | PS `abilities.js` | Ability data: generation introduced |
| `formats.json` | Manually curated | 32 competitive format definitions (Gen 1–9 general + 22 mainline game formats) |
| `learnsets-g6-allowlist.json` | Derived | Gen 6 legality cross-reference for transfer moves |
| `version.json` | Computed | SHA-256 hash of each file above; compared by `FormatService` to detect when backend data has been updated |

`FormatService` loads these files at startup (from Hive cache if available, otherwise from the bundled assets), then checks `GET /ps-data/version` in the background. If any SHA differs, it downloads the updated file from `GET /ps-data/file/:name` and saves it to Hive.

To update the PS data:

```bash
pip install requests json5
python scripts/sync_ps_data.py
git add assets/data/ps/ backend/app/static/
git commit -m "chore: update PS data"
# Redeploy backend — clients detect the SHA change and auto-download
```

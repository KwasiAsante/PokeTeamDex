# scripts/

Maintenance scripts. Run locally by the developer — not part of the Flutter build or CI pipeline.

---

## sync_ps_data.py

Downloads, trims, and bundles Pokémon Showdown data for both the Flutter app and the backend — `shared/ps_data/` at the project root is the single source of truth for both (no more separate `assets/data/ps/` + `backend/app/static/` copies).

### What it does

1. Fetches learnsets, the pokedex, and format tier data straight from the PS TypeScript source (`data/learnsets.ts`, `data/mods/gen{N}/learnsets.ts`, `data/pokedex.ts`, `data/mods/gen{N}/pokedex.ts`, `data/formats-data.ts`) rather than the old lossy compiled `learnsets.json` endpoint; moves/items/abilities still come from PS's compiled JSON (`moves.json`, `items.js`, `abilities.js`) since their TS source embeds JS callbacks the script can't evaluate
2. Derives per-generation learnset files directly from the TS source (`via_prevo` pre-resolved, no prevo-chain traversal needed downstream)
3. Trims fields the app doesn't need (reduces file sizes significantly)
4. Writes processed JSON to `shared/ps_data/`
5. Computes a SHA-256 hash per file, written into `shared/ps_data/version.json`

### Output files

| File | Description |
|------|-------------|
| `shared/ps_data/learnset_1.json` … `learnset_9.json` | Per-generation, per-Pokémon move lists (method, level, `via_prevo`) |
| `shared/ps_data/pokedex.json` | Species/variety base data |
| `shared/ps_data/pokedex-gen-overrides.json` | Per-gen stat/type/ability overrides for Pokémon that differ from their Gen 9 defaults |
| `shared/ps_data/moves.json` | Move stats (type, power, accuracy, PP, Z/Max data) |
| `shared/ps_data/items.json` | Item data (gen introduced, Mega/Z mappings) |
| `shared/ps_data/abilities.json` | Ability data (gen introduced) |
| `shared/ps_data/formats-data.json` | Per-format tier/banlist data |
| `shared/ps_data/version.json` | SHA-256 of each file above; polled by both `FormatService` (Flutter) and `GET /ps-data/version` (backend) |

`assets/data/ps/formats.json` (32 competitive format definitions) is separate — it's manually curated, not generated by this script, and still lives under `assets/data/ps/` since it isn't PS-sourced data. Everything else previously under `assets/data/ps/` or `backend/app/static/` is a frozen pre-migration snapshot; don't edit or read from those paths going forward.

### Usage

```bash
# Install dependencies (one-time)
pip install requests json5

# Run
python scripts/sync_ps_data.py

# Commit the output
git add shared/ps_data/
git commit -m "chore: update PS data"
```

### When to run

- After a new Pokémon generation or DLC releases (new Pokémon / moves / items)
- When Pokémon Showdown updates their format definitions
- After catching a learnset validation bug (stale data)

After committing and deploying the backend, any app instance that fetches `GET /ps-data/version` will detect the SHA change and download the updated files automatically in the background.
